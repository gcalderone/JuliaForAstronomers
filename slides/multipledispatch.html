Multiple dispatch

  • Single dispatch (OOP)
  • (C++, Java, Python, etc...)
  • Methods belong to data
  • Multiple dispatch
  • (Julia, Haskell, etc...)
  • Methods and data are separate

OOP + method overload + all class members are static $\equiv$ multiple dispatch

Grammar analogy

  • Focus on subjects ($\equiv$ class)
  • All verbs ($\equiv$ methods) can only be applied to a single subject, e.g.
    • person->walk()
    • socket->send(data)
    • number1->add(number2)
    • etc...
  • Focus on actions ($\equiv$ methods)
  • All verbs can be applied to any number of subjects, e.g.
    • walk(person)
    • send(socket, data)
    • add(num1, num2, num3, ...)
    • etc...
